# Verification Errors - 2026-06-15

## Generated project verification working directory

- Generated frontend verification initially ran npm from `backend/` instead of the generated frontend directory.
- Maven tool discovery later ran inside the generated backend and could not import the factory's `app.workflow`.
- Resolution: run package commands in the generated project directory and set `PYTHONPATH` to the factory backend only when factory tooling is needed.

## Direct Maven used Java 8

- Direct Maven execution inherited Java 8 and reported class version `61.0 / 52.0`.
- Resolution: use the same `_maven_subprocess_env()` as the production workflow so generated Spring Boot 3 projects run under JDK 17.

## In-app browser disconnected

- Browser regression could not continue because the in-app browser session returned no active tabs.
- Static frontend build and generated frontend build passed; visual smoke testing should be repeated after reconnecting the browser.

## PowerShell Start-Process log redirection

- `Start-Process` rejects using the same path for `RedirectStandardOutput` and `RedirectStandardError`.
- Resolution: assign separate stdout and stderr log files for each background development service.

## Windows PowerShell command separators

- Windows PowerShell 5 does not accept Bash-style `|| exit 0`.
- Resolution: run `rg` directly and interpret exit code 1 as “no matches”, or use a native PowerShell `if` statement.
